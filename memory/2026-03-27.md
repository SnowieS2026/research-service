## Key Technical Findings
- Chroma v2 uses HTTP not HTTPS for localhost — vector-store.ts already correct
- Ollama embed endpoint: POST body must be plain JSON object `{model, prompt}` — not JSON string
- OAuth 1.0a signing: sig base string does NOT include POST body params, only OAuth header params sorted alphabetically
- X API PKCE: needs `code_verifier` (random 64-char base64url) + `code_challenge` (sha256 of verifier, base64url)
- X API OAuth 2.0 client_credentials: requires `client_type=third_party_app` or `service_client` in body
