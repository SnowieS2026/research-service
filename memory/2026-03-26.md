## 🚨 Critical Bug Found — AJ05RCF Advisory Miss

**What:** Old vehicle-osint.py pipeline was missing 26 of 27 MOT advisories.

**Root cause:** `estimateAdvisories()` used a hardcoded keyword dictionary (~50 entries). If a MOT term wasn't in the dictionary, it silently returned no match. The Mondeo's brake pipes, suspension bushes, multiple coil springs — none were in the keyword list.

**What I told Infinitara last night (lost to compaction):** "Only needs new tyres at £50-80, clean car, scope for £500 profit" — WRONG.

**What the full MOT text shows:** 27 advisories across 19 tests, including:
- Brake pipe corroded (safety critical)
- Subframe bush worn (x3)
- Coil spring corroded (x4)
- Brake hose deteriorated (x5)

**Advisory cost reality:** £1,220–£5,440 estimated repair costs.

**This could have bankrupted Infinitara** before even starting bounty work. He had arranged a viewing and was planning to buy based on my bad advice.

**Lesson:** Always parse the actual MOT text directly — never rely on a keyword dictionary against raw text when the MOT has structured data. The new parser splits each MOT test block individually and extracts every `Advice` line.

## Edit Tool — `~` Path Failures

**Problem:** Using `~` in file paths caused silent failures in the edit tool on Windows. The tool received `~\.openclaw\workspace\...` but couldn't resolve it.

**Root cause:** `~` is PowerShell's `$HOME` alias, but edit tool runs outside PowerShell process context. `HOME` env var was set to `C:\Users\bryan` but the tool's path resolution didn't use it.

**Fix applied:**
- Added to Red Lines in AGENTS.md: "Never use `~` in file paths"
- Updated USER.md known issues: "Never use `~` — use full absolute paths"
- Committed: `44dde3bc`

**Going forward:** Always use `C:\Users\bryan\.openclaw\workspace\...` — never `~`.
