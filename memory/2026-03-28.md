Root cause: `parseSignals()` in `dashboard.ts` used `extract()` helper which stripped `###` heading markers before parsing. Signals section uses `###` headings (from GLM-5 writer) but parser expected `**bold**` format.
Fix: `parseSignals()` now iterates raw markdown lines directly, detecting `### ` title lines and blank-line-separated content paragraphs.

Second bug: SEO `<!--` metadata block was being absorbed into the last signal's content (signal 3 ended with keywords/description instead of proper ending).
Fix: parser now stops at `<!--` opening comment line (in addition to standalone `---` divider).

Third bug: BeeHiiv formatter had `s.content.slice(0, 200)` truncating each signal to 200 chars with `...`.
Fix: removed the slice -- full content now renders.
